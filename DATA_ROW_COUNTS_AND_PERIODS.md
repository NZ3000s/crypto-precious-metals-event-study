# Кількість рядків і періоди даних / Row counts and data periods

Детальний опис обсягу даних та часових діапазонів по кожному джерелу та фінальним аналітичним файлам. Вимога курсу **≥10 000 спостережень** виконується за рахунок **1-хвилинного** датасету (`features_1m.csv`, 53k+ рядків); щоденна та погодинна панелі використовуються для event study та описової статистики.

---

### 1.3 1-хвилинний датасет (≥10 000 спостережень)

**Файл:** `data/processed/features_1m.csv`

| Показник | Значення |
|---------|----------|
| **Кількість рядків** | **53 864** (одна хвилина = один запис по XAUUSDT) |
| **Період** | 2026-01-05 00:00 — 2026-02-11 09:43 (UTC) |
| **Призначення** | Задоволення вимоги курсу **≥10 000 записів**; Binance 1m + щоденні ETF/контролі по даті. |

**Подія (event = launch):** 2026-01-05 — запуск Binance XAUUSDT TRADFI perpetual.

- **Pre-event (до події):** дні з 2025-09-02 по 2026-01-04 (тільки ETF + контролі; даних Binance ще немає).
- **Post-event (після події):** дні з 2026-01-05 по 2026-02-10 (з’являються дані Binance по золоту та, з запізненням, по сріблу).

Точну кількість днів pre/post можна побачити в аналізі: `df_core["post_event"].value_counts()`.

**Вимога курсу ≥10 000 записів:** виконується датасетом **`features_1m.csv`** (53 864 рядки). Щоденна панель (112) використовується для event study; погодинна (3 888) — опційно для вищої частоти.
